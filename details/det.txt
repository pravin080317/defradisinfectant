## ✅ ISSUE 1: **“View supporting documents” / “Add documents” links lack context**

### **Problem:**
Multiple rows have the same link text:  
- “View supporting documents”  
- “Add documents”  

These **do not indicate which report they apply to**, making it inaccessible for screen reader users.

---

### ✅ Fix: Add visually hidden context to each action link

#### Find this block:

```liquid
{% if attachments.results.entities.size == 0 %}
  <a class="govuk-link" tabindex="0" href="{{ sitemarkers['UMR004 Upload Evidence'].url }}?id={{ result.voap_portalrequestid }}">
    Add documents
  </a>
{% else %}
  <a class="govuk-link" tabindex="0" href="{{ sitemarkers['UMR006 Report Details'].url }}?id={{ result.voap_portalrequestid }}#supporting-documents">
    View supporting documents
  </a>
{% endif %}
```

---

### ✅ Replace with:

```liquid
{% if attachments.results.entities.size == 0 %}
  <a class="govuk-link" tabindex="0" href="{{ sitemarkers['UMR004 Upload Evidence'].url }}?id={{ result.voap_portalrequestid }}">
    Add documents<span class="govuk-visually-hidden"> for report {{ result.voap_billingauthorityreportnumber }}</span>
  </a>
{% else %}
  <a class="govuk-link" tabindex="0" href="{{ sitemarkers['UMR006 Report Details'].url }}?id={{ result.voap_portalrequestid }}#supporting-documents">
    View supporting documents<span class="govuk-visually-hidden"> for report {{ result.voap_billingauthorityreportnumber }}</span>
  </a>
{% endif %}
```

## ✅ ISSUE 2: **Table header sort buttons lack `aria-label`s**

Same as before, the column sorting buttons are good visually but need accessible labels.

---

### ✅ Fix:

Update each `<button>` inside `<th>` to include a `aria-label`.

#### Example:
```html
<button type="button" onclick="sortTable(0)" class="govuk-link" style="color:#005ea5"
        aria-label="Sort by billing authority report number">
  Billing authority report number
</button>
```

Apply this to:

| Column | `aria-label` |
|--------|--------------|
| Billing authority report number | `Sort by billing authority report number` |
| Billing authority reference number | `Sort by billing authority reference number` |
| Date created | `Sort by date created` |
| Postcode | `Sort by postcode` |

## ✅ ISSUE 3: **Pagination links need `aria-labels`**

The pagination links (1, 2, 3… Next) don’t describe their purpose.

---

### ✅ Fix in your JS: Add `aria-label` in `addPaginationLink(i)`

In `VOA/GDS_Table_Sorting_Pagination_JS`, update:

```js
link.setAttribute('aria-label', 'Go to page ' + i);
```

Also update the **Next** and **Previous** buttons:

```html
<span class="govuk-pagination_link-title" aria-label="Go to next page">Next</span>
<span class="govuk-pagination_link-title" aria-label="Go to previous page">Previous</span>
```
