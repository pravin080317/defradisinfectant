## ✅ ISSUE 1: **“Edit” link lacks context for screen reader users**

### **Problem:**
Multiple `Edit` links are present in the table, but they don’t describe what’s being edited. A screen reader would just say “Edit” with no idea which row or report it relates to.

---

### ✅ Fix:

#### In your current markup:

```liquid
<a class="govuk-link" tabindex="0" href="{{ sitemarkers['FE006 Check and Modify Your Report Details'].url }}?id={{ result.voap_portalrequestid }}"> Edit </a>
```

#### Replace it with:

```liquid
<a class="govuk-link" tabindex="0"
   href="{{ sitemarkers['FE006 Check and Modify Your Report Details'].url }}?id={{ result.voap_portalrequestid }}">
   Edit<span class="govuk-visually-hidden"> report for billing authority {{ result.voap_billingauthorityreportnumber }}</span>
</a>
```

> **Outcome:** For screen reader users, this will now read:  
> “Edit report for billing authority 20230004”

## ✅ ISSUE 2: **Table column headers with sort buttons lack `aria-label`**

Each `<th>` contains a `<button>` to sort the table, but there is no screen reader label explaining what the button does.

### Fix for sorting buttons:

#### Before:
```html
<button type="button" onclick="sortTable(0)" class="govuk-link" style="color:#005ea5">
  Billing authority report number
</button>
```

#### After:
```html
<button type="button" onclick="sortTable(0)" class="govuk-link" style="color:#005ea5"
        aria-label="Sort by billing authority report number">
  Billing authority report number
</button>
```

Repeat this for the “Billing authority reference number” column too:

```html
<button type="button" onclick="sortTable(1)" class="govuk-link" style="color:#005ea5"
        aria-label="Sort by billing authority reference number">
  Billing authority reference number
</button>
```

## ✅ ISSUE 3: **Pagination links lack `aria-label` and clarity**

The pagination controls at the bottom have numeric links like:

```html
<a class="govuk-link govuk-pagination_link" href="#">1</a>
```

This is ambiguous to screen readers.

---

### Fix (in `addPaginationLink(i)` in JS):

Add this line:
```javascript
link.setAttribute('aria-label', 'Go to page ' + i);
```

So it becomes:
```javascript
link.textContent = i;
link.setAttribute('aria-label', 'Go to page ' + i);
```

Also add:
```html
<span class="govuk-pagination_link-title" aria-label="Go to next page">Next</span>
```

And similarly for **Previous**.
